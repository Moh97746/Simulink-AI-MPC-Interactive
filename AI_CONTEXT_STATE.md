# دليل الذاكرة والسياق للذكاء الاصطناعي (AI Context & State Memory)
**المشروع:** Simulink-to-Interactive-Web Skill  
**الهدف من هذا الملف:** توفير السياق الكامل، الهيكلية، والقواعد الصارمة لأي محادثة مستقبلية مع وكيل ذكاء اصطناعي (AI) لتجنب الأخطاء الكارثية السابقة ومواصلة التطوير باحترافية.

---

## 🏗️ 1. الهيكلية المعمارية (Architecture)
المشروع ليس مجرد موقع، بل هو **"مهارة (Skill)"** برمجية لتحويل ملفات السيمولينك إلى واجهات ويب تفاعلية. يتكون من 3 ملفات بايثون أساسية وملف HTML ناتج:

1. `slx_parser.py`: (المُستخرج) يقرأ ملفات `.slx` (وهي في الواقع ملفات ZIP تحتوي على XML) ويستخرج البلوكات والوصلات ليولد كود بايثون مبدئي للرسم.
2. `draw_light.py`: (مكتبة الرسم) تحتوي على دوال مثل `draw_block` و `draw_ortho_arrow`. تعتمد على `matplotlib` لتوليد صور SVG فائقة الدقة.
3. `build_interactive.py`: (المُجمّع أو المنسج) أهم ملف. يقوم بقراءة الـ SVGs الناتجة، ودمجها داخل قالب HTML يحتوي على CSS و JavaScript للواجهة التفاعلية والمحاكاة.
4. `index.html`: النتيجة النهائية (تطبيق ويب بملف واحد).

## 🚫 2. الأخطاء القاتلة السابقة (التي يجب عدم تكرارها أبداً)

### ❌ الكارثة الأولى: صراع أقواس `{}` بين Python و JavaScript
يستخدم ملف `build_interactive.py` دالة `.format()` في بايثون لضخ ملفات SVG وقاموس الشروحات داخل قالب الـ HTML.
* **المشكلة:** دالة `.format()` تقرأ أي قوسين `{ }` في النص على أنها متغيرات بايثون. وبما أن القالب يحتوي على كود JavaScript و CSS مليء بالأقواس `{}`، فإن بايثون ينهار ويُصدر خطأ `KeyError` أو `ValueError`.
* **الحل الدائم والشرط الصارم:** **كل أقواس جافاسكربت و CSS في القالب يجب أن تكون مزدوجة `{{ }}`**.
  * كود JS خطأ: `function test() { return {x: 1}; }`
  * كود JS صحيح: `function test() {{ return {{x: 1}}; }}`
  * الأقواس الوحيدة المفردة في القالب بأكمله هي: `{SVG_ROOT}`, `{SVG_AI}`, `{SVG_WT}`, `{EXPLANATIONS_JSON}`.

### ❌ الكارثة الثانية: استخدام Regex لتعديل الأكواد
في مرحلة سابقة، قام الـ AI بكتابة سكربت منفصل `run_refactor.py` يستخدم التعابير القياسية (Regex) لتعديل محتوى `build_interactive.py` برمجياً.
* **النتيجة:** دمر السكربت بنية الملف، وخلط بين الأقواس المزدوجة والمفردة، وترك بقايا أكواد يتيمة.
* **الشرط الصارم:** **يُمنع منعاً باتاً** استخدام سكربتات خارجية تعتمد على Regex لإجراء تعديلات على ملف `build_interactive.py`. أي تعديل يجب أن يتم عبر استبدال الأسطر المباشر (Line-by-line replacement) باستخدام أدوات التعديل الخاصة بالذكاء الاصطناعي.

## ⚙️ 3. الميزات الحالية وكيفية عملها (Current Features)

1. **دعم اللغتين (i18n):** 
   - تم بناء قاموس في كود الـ JS `window._L`.
   - دالة `toggleLanguage()` تغير كل نصوص الواجهة (الأزرار، التبويبات، الشرح الجانبي، ونصوص مراحل المحاكاة) دون تحديث الصفحة.
   - **ملاحظة:** اتجاه الصفحة ثابت دائماً من اليمين لليسار (RTL).
2. **المحاكاة (Simulation):**
   - تعتمد على نظام `setTimeout` مكدسة داخل مصفوفة `simIntervals`.
   - متصلة بكاميرا `flyTo(x, y, scale)` لعمل تتبع تلقائي (Auto-Tracking).
   - يتم تغيير سرعة المحاكاة عبر متغير `dt` بناءً على قيمة شريط التمرير `#simSpeed`.
3. **تصدير 8K PNG:**
   - يتم سحب الـ SVG، وقراءة الـ `viewBox` الأصلي، وضربه في 4 (`* 4`)، ثم رسمه على `canvas` لتحميله بصيغة PNG بدقة فائقة للأوراق العلمية.
4. **الوضع الليلي (Dark Mode):**
   - يعتمد على كلاس `body.dark`. يتم تبديله عبر checkbox في مركز التحكم، ويعدل ألوان الـ CSS لتصبح من درجات الـ Slate الداكنة.

## 📝 4. قواعد التطوير والإضافة لأي AI في المستقبل

إذا طلب منك المستخدم **إضافة ميزة جديدة**، اتبع هذا التسلسل الصارم:

1. **الاستكشاف أولاً:** استخدم أداة `view_file` لقراءة السطور المعنية في `build_interactive.py` قبل كتابة أي سطر كود.
2. **التعديل الموضعي:** استخدم أداة `replace_file_content` (أو multi) وحدد أسطر البداية والنهاية بدقة متناهية.
3. **التأكد من الأقواس:** إذا كنت تضيف كود JavaScript أو CSS داخل القالب، اسأل نفسك مرتين: *"هل ضاعفت الأقواس لتصبح `{{ }}`؟"*
4. **ربط اللغات:** إذا أضفت زراً جديداً أو نصاً جديداً، يجب إضافته في نسختين (عربي وإنجليزي) داخل قاموس `const L` في دالة `toggleLanguage()`.
5. **معرفات الـ SVG (GIDs):** أي بلوك أو سهم جديد في كود الرسم (`draw_light.py`) يجب أن يأخذ مُعرف `gid="block_name"`. هذا المُعرف هو الرابط السري بين الرسمة وبين الشرح في كود الجافاسكربت.
